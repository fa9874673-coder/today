# AGENTS.md

本文件用于约束 Codex 在本项目中的工作方式。适用于个人 GitHub 项目、网页项目和 Python 学习项目。

## 开工规则

- 每次开始任务前，必须先只读分析项目结构、关键文件和 Git 状态。
- 修改文件前，必须先说明计划、影响范围和风险，并等待用户确认。
- 不确定时必须提问，不要猜测。

## 文件安全

- 不得擅自删除、移动、覆盖重要文件。
- 不得上传、提交或展示 `.env`、token、密钥、私人邮箱、cookie、浏览器数据等敏感信息。
- 不得擅自修改 `.gitignore`、`README`、GitHub Pages 配置，除非任务明确要求。

## Git 规则

- 不得擅自 `commit`、`push`。
- 不得擅自创建、删除、重建远程仓库。
- 提交前必须展示改动摘要、风险点和待提交文件。
- 检查 Git 邮箱是否可能暴露私人邮箱，但不要完整输出私人邮箱。

## 修改后规则

- 必须说明改了哪些文件。
- 必须说明为什么这样改。
- 必须给出运行、测试或验收方法。
- 网页项目应尽量用 browser 插件预览；如果无法预览，必须说明原因。

## 限制说明

- 如果遇到权限、沙箱、网络或环境限制，必须明确说明。
- 不要把无法检查的内容当成已经检查。

## 每次开工标准流程

1. 只读分析项目结构、关键文件和 Git 状态。
2. 汇报项目类型、风险点和初步判断。
3. 提出修改计划并等待用户确认。
4. 仅在确认后修改必要文件。
5. 修改后说明文件变化和原因。
6. 给出测试、运行或验收方法。
7. 不提交、不推送，除非用户明确要求。
